## Safety Defaults

- Read-only verbs such as `list`, `get`, `show`, `messages`, and `service-info` are allowed by default.
- Treat verbs such as `create`, `edit`, `delete`, `reboot`, `reinstall`, `reset`, `terminate`, `reply`, `switch`, and `set-*` as state-changing until proven otherwise.
- When uncertain, inspect the command with `--help` first rather than running it.

## Global Workflow

1. Run `ovhcloud --help` or `ovhcloud <command> --help` to map the command tree.
2. Confirm auth and profile context before the first authenticated call.
3. Prefer `-o json` or `-o yaml` for scripting and reproducible parsing.
4. Use gval output expressions or `--filter` only after you have inspected the raw fields once.

## Auth And Profiles

- Config files are resolved from `./ovh.conf`, `~/.ovh.conf`, then `/etc/ovh.conf`.
- Profile precedence is `--profile`, then `OVH_PROFILE`, then the `[default]` config entry, then legacy mode.
- Environment-based auth may use either application credentials or OAuth client credentials, depending on the account setup.

## Important Flags

- `--profile <name>` selects a profile for a single invocation.
- `-o json` and `-o yaml` are the safest automation outputs.
- `-o '<expr>'` and `--filter '<expr>'` use gval expressions; inspect fields before relying on them.
- `--debug` is useful for troubleshooting but may expose sensitive request details in logs.
