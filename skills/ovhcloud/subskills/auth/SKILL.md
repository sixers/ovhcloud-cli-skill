# ovhcloud-auth

## When To Use

- You need to explain how `ovhcloud` authenticates or which credentials source wins.
- You need to use `ovhcloud login`, environment variables, or `.ovh.conf` safely.
- You want a minimal authenticated read-only smoke test before opening a service-specific skill.

## Start Here

- Read [ovhcloud-shared](../shared/SKILL.md) first.
- Open the [generated auth reference](../../references/generated/auth.md) for the exact config keys and precedence rules.
- Prefer a read-only smoke test such as `ovhcloud account get -o json` after auth is confirmed.

## Read-Only First

- Explaining or inspecting authentication is safe.
- `ovhcloud login` itself changes local credentials but does not mutate OVHcloud resources; still confirm before using it in a shared machine context.
- Avoid mutating service commands while validating credentials; use `account get`, `cloud project list`, or another read-only command instead.

## Important Identifiers And Flags

- Application-key auth uses `OVH_APPLICATION_KEY`, `OVH_APPLICATION_SECRET`, and `OVH_CONSUMER_KEY`.
- OAuth client auth uses `OVH_CLIENT_ID` and `OVH_CLIENT_SECRET`.
- `OVH_ENDPOINT` selects the API endpoint, and `--profile` or `OVH_PROFILE` selects the named profile.
